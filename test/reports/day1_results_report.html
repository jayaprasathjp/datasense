Testing the execution time of the code successfully generated by the Gemma-4 DataSense model across all 4 core tasks. Note: LLM generation time (~30-40s) is identical for both; we are measuring the execution bottleneck.

04 / Autonomous Self-Debugging Loop

During the LLM-in-the-loop tests, the DataSense model hallucinated incorrect Pandas syntax twice. Our custom autonomous loop caught the exceptions, formulated targeted feedback, and prompted the LLM to fix itself.

  • Time-Series Alerting: The model attempted .abs() on a SeriesGroupBy object. The loop fed the AttributeError back. The LLM corrected the syntax on Attempt 2 resulting in a successful run.
  • Priority Ranking: The model missed a markdown code block on Attempt 1. The loop detected the parsing failure and recovered on Attempt 2.
  • Executor Safety: We implemented sandbox restrictions specifically blocking pd.read_csv and trapping ModuleNotFoundError for case-sensitive errors (e.g. cuDF vs cudf).
